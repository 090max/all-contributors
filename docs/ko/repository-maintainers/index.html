리포지토리 관리자를위한 참고 사항

기여자는 자신의 노력이 보장 될 수 있도록 나중에 목록이 업데이트 될 때까지 기다리지 않고 기여 발견 즉시 바로 인정되어야 합니다. Repo에 커밋 한 결과가 아닌 기여에 대해서는 어려울 수 있지만, 기여와 목록 업데이트 사이의 시간을 최소화 해 주세요.

프로 팁: 프로젝트가 @all-contributors 🤖 봇을 사용할 경우 Issue나 Pull Request에 코멘트를 남겨서 공로를 인정해주세요. 예:

@all-contributors please add @tbenning for design ✨

리포지토리의 소유자 또는 관리자라 하여도 기여자 목록을 최신 상태로 유지하는 것은 전적으로 귀하의 책임은 아닙니다. 기여자가 자신을 목록에 추가할 수 있도로 권장해주셔야 합니다. Issue, 블로그, 질문 응답 형태나 적절한 경우, 더 직접적인 방법을 통해 이루어질 수 있습니다.

많은 기여자들은 자신이 한 일이 기여자 명단에 들어 갈만큼 가치가 있다는 것을 잘 모를 것입니다. 기여자 사양을 읽지 않거나 자신이 한일이 중요한 일이 아니라고 생각하기 때문일 것 입니다. 이럴 경우, 자신이 직접 기여자 명단에 추가할 수 있도록 하는게 좋지만, 그렇지 못할경우에는 자신이 직접 기여자를 추가 해주시는게 좋습니다 (Pull Request 형태가 좋습니다).

어떤 사람이 기여자로 인정돼야 하는가에 대한 완전한 규칙은 없지만 모든 공헌자가 인정될 수 있도록 노력 해주세요.
